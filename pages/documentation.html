Documentation guide

This document explains you how to contribute to document/user guide preparation of Music Blocks.


Mainly there are two methods which you can use to improve/contribute to the documentation aspects of Music Blocks. These were designed by the fact that there are two main user categories who has some technical background where others have no technical background at all. What is meant by "technical" is they know about html and CSS.

1. With no technical background


For these user category we've implemented a very simple way of documentating with "Readme.md" files. Users first need to get a fork from the existing repo, and then modifications can be done in appropriate readme files. Once done, can send a pull request to the original repo. Once the modification is reviewed and validated, content of the readme.md file wll be taken into the website.
